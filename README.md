# mini-spectrum-maze-reboot

Three.js로 구현한 빛의 색에 따라 길이 변화하는 3D 미로 퍼즐 게임

## 3D 미로 퍼즐 게임 (Three.js)

---

## 1. 프로젝트 개요

### 1.1 프로젝트 목표

-   순수 JavaScript + Three.js를 이용한 3D 퍼즐 게임 제작
-   단일 스테이지 완성
-   플레이어가 출발점에서 도착점까지 이동하여 클리어
-   일반적인 게임 개발 흐름 학습

### 1.2 핵심 컨셉

-   캐릭터는 **빛을 사용해 미로의 길을 확인**
-   **빛의 색에 따라 보이는 길이 달라짐**
-   플레이어는 빛의 색을 전환하며 미로를 탐색

### 1.3 참고 게임

-   _Monument Valley_
    -   고정 시점 (아이소메트릭/쿼터뷰)
    -   단순하지만 직관적인 퍼즐 구조

## 3. 단계별 개발 진행

---

## 3.1 기획 단계 (사전 설계)

### 목표

-   구현 전에 게임의 규칙과 범위를 명확히 정의

### 할 일

-   게임 플레이 흐름 정의
    -   시작 → 이동 → 도착 → 클리어
-   빛 컨셉 정의
    -   빛 색 종류 (예: White / Blue)
    -   색에 따라 보이는 길의 규칙
-   플레이 방식 결정
    -   키보드 이동 또는 클릭 이동
-   프로젝트 범위 제한
    -   단일 스테이지만 제작

📌 **이 단계에서는 실제 구현 없이 문서 정리만 진행**

---

## 3.2 기본 게임 뼈대 구현

### 목표

-   퍼즐 요소 없이도 게임이 끝까지 플레이 가능하도록 구성

### 구현 항목

-   Three.js 기본 씬 구성
    -   Scene / Camera / Renderer
-   고정 카메라 시점 구현
    -   아이소메트릭 또는 쿼터뷰
-   미로 구조 배치
    -   바닥 타일
    -   벽 또는 이동 불가능 영역
-   플레이어 캐릭터 구현
    -   기본 형태 (큐브 등)
    -   이동 로직 구현
-   시작점 & 도착점 설정
-   클리어 조건 처리
    -   도착점 도달 시 클리어 출력

🎯 **이 단계 완료 기준**

-   퍼즐 요소 없이도 출발 → 이동 → 클리어 가능

---

## 3.3 최소 기능 버전 (MVP) 완성

### 목표

-   게임으로서 최소한의 완성도 확보

### 구현 항목

-   이동 제한 처리
    -   벽 충돌
    -   이동 불가 영역
-   간단한 UI
    -   현재 상태 표시 (예: "Clear")
-   코드 구조 정리
    -   Player
    -   Maze
    -   Game Loop

📌 이 단계까지는 **빛 컨셉 미적용**

---

## 3.4 핵심 컨셉 구현 (빛 & 색상)

### 목표

-   핵심 퍼즐 메카닉 적용

### 구현 항목

-   빛 색상 시스템
    -   현재 활성화된 빛 색 변수
-   타일 속성 추가
    -   `visibleColor` 또는 `requiredLightColor`
-   색상에 따른 가시성 처리
    -   조건 불일치 시 타일 숨김 또는 투명 처리
-   빛 전환 입력
    -   키 입력 또는 버튼

🎯 **이 단계 완료 기준**

-   빛의 색을 바꾸면 보이는 길이 달라짐
-   퍼즐이 성립함

# TS + React 전환 이점 정리 (KO)

## 1) TypeScript 전환 장점

### 컴파일 타임 오류 탐지
- 런타임에서 뒤늦게 발견되던 타입 불일치를 빌드/에디터 단계에서 조기 발견 가능
- `Direction`, `TileColor`, `MazeMap` 같은 핵심 도메인 타입을 계약으로 고정해 실수 전파를 줄임

### 리팩터링 안전성
- 함수 시그니처/타입 변경 시 영향 범위를 IDE와 타입체커가 추적
- 엔진 내부 구현 변경 시에도 공개 인터페이스 준수 여부를 빠르게 검증 가능

## 2) React 전환 장점

### 상태/이벤트 관리 일원화
- `cleared`, `selectedColor`, `speed`, `angle` 같은 UI 상태를 React state에서 일관 관리
- 버튼/키 입력/슬라이더 이벤트를 단일 컴포넌트 트리에서 제어해 흐름 파악이 쉬움

### UI 재사용성
- `GameUI`, `DirectionPad`, `ColorButtons`, `Toast`를 컴포넌트 단위로 분리하여 재조합 가능
- 엔진 로직과 UI를 분리해 화면 개선이 게임 코어에 미치는 영향을 최소화

## 3) 팀 관점 장점

### 역할 분리
- 엔진 담당은 `GameEngine` 및 물리/렌더/입력 규칙에 집중
- UI 담당은 React 컴포넌트/스타일/상태 표현에 집중

### 병렬 작업
- 엔진 API 계약이 고정되면 UI와 엔진을 동시에 개발/검증 가능
- 충돌 지점이 줄어 브랜치 통합 비용 완화

### 코드리뷰 용이성
- 타입 정의와 컴포넌트 경계가 명시되어 변경 의도가 코드에 드러남
- “UI 변경인지, 엔진 규칙 변경인지”를 리뷰 단계에서 빠르게 분류 가능
